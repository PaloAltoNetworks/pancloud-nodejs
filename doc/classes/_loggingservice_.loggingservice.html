poll

  • poll(qid: string, sequenceNo: number, maxWaitTime?: undefined | number): Promise<jobResult>
    • Defined in loggingservice.ts:234

    Used for synchronous operations (when the auto-poll feature of a query is not used)

    Parameters

    • qid: string

      the query id to poll results from

    • sequenceNo: number

      This number begins at one more than the sequence number returned when you initially create the query, and it must monotonically increase by 1 for each subsequent request. It is permissible to re-request the current sequence number. However, attempts to decrease the sequence number from one request to the next, or to increase this number by more than 1, will result in an error

    • Optional maxWaitTime: undefined | number

      Maximum number of milliseconds you want the HTTP connection to the Logging Service to remain open waiting for a response. If the query cannot be completed in this amount of time, the service closes the HTTP connection without returning results. Either way, to obtain complete query results your application must continue to request result sequences until this API reports either JOB_FINISHED or JOB_FAILED. This parameter's maximum value is 30000 (30 seconds). If this parameter is not specified, 0 is used, in which case the HTTP connection is closed immediately upon completion of the HTTP request

    Returns Promise<jobResult>

    a promise with the Application Framework response

query

  • query(cfg: lsQuery, eCallBack?: function | null, sleep?: undefined | number, fetchTimeout?: undefined | number): Promise<jobResult>
    • Defined in loggingservice.ts:178

    Performs a Logging Service query call and returns a promise with the response. If the "eCallBack" handler is provided then it will be registered into the event topic and this query will be placed into the auto-poll queue (returned events will be emitted to the handler)

    Parameters

    • cfg: lsQuery

      query configuration object

    • Optional eCallBack: function | null

      toggles the auto-poll feature for this query and registers the handler in the 'event' topic so it can receive result events. Providing 'null' will trigger the auto-poll feature for the query but without registering any handler to the 'event' topic (to be used when a handler is already registered to receive events)

    • Optional sleep: undefined | number

      if provided (in milliseconds), it will change this Logging Service object auto-poll delay value (the amount of time between consecutive polls). Please note that this may affect other queries already in the auto-poll queue

    • Optional fetchTimeout: undefined | number

      milliseconds before issuing a timeout exeception. The operation is wrapped by a 'retrier' that will retry the operation. User can change default retry parameters (3 times / 100 ms) using the right class configuration properties

    Returns Promise<jobResult>

    a promise with the Application Framework response
